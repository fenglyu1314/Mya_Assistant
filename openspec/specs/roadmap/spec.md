# 开发路线图（Roadmap）

> 基于技术方案 v1.2，定义各阶段的目标、交付物和验收标准。
> 具体实现任务由 openspec change 管理，本文档不包含 task 级别的细节。

---

## Phase 0：项目搭建 + 后端接入

**状态**：`done`
**前置依赖**：无

**目标**：完成基础设施搭建，验证核心技术可行性。

**核心交付物**：
- pnpm monorepo 项目结构（`shared` / `mobile` / `desktop` 三个包）
- Supabase 项目创建 + 数据库表结构（schedules / todos / notes）+ RLS 策略
- 共享层基础结构：数据模型类型定义 + SyncAdapter 接口 + SupabaseAdapter 实现
- 移动端 SDK 接入验证（CRUD + Realtime 订阅）

**验收标准**：
- monorepo 结构就绪，`pnpm install` 正常运行，包间依赖正确解析
- 移动端能通过 SyncAdapter 对 Supabase 执行增删改查
- Realtime 订阅正常工作（一端写入，另一端实时收到变更）
- 所有表均已配置 RLS 策略，未认证请求被拒绝

---

## Phase 1：基础骨架

**状态**：`done`
**前置依赖**：Phase 0

**目标**：完成用户认证 + 第一个可用功能模块（快速记录），建立端到端开发闭环。

**核心交付物**：
- 移动端项目骨架（导航框架、主题系统、基础 UI 组件）
- Supabase Auth 集成（邮箱登录）
- 快速记录模块完整功能（创建、查看、删除、置顶、分类筛选、云端实时同步）

**验收标准**：
- 用户可通过邮箱注册/登录
- 登录后能在移动端创建/查看/删除/置顶快速记录
- 数据持久化到 Supabase，刷新后数据不丢失
- 未登录状态下无法访问数据

---

## Phase 2：核心功能 + 离线同步

**状态**：`planning`
**前置依赖**：Phase 1

**目标**：完成待办和日程两大核心模块，建立离线优先的同步基础。

**核心交付物**：
- 待办事项模块（CRUD、优先级、截止日期、完成状态、子任务、拖拽排序）
- 日程管理模块（日历视图、日程 CRUD、本地提醒通知）
- 离线支持引擎（本地 SQLite 缓存、断网操作队列、网络恢复自动同步、字段级冲突合并）

**验收标准**：
- 待办和日程的完整 CRUD 可用，数据与后端实时同步
- 待办支持子任务嵌套和拖拽排序
- 日程支持日历视图展示和本地通知提醒
- 断网状态下可正常操作，联网后自动同步且字段级合并无数据丢失
- 两端同时修改同一记录的不同字段时，合并结果正确

---

## Phase 3：桌面端 + 电纸书端适配

**状态**：`planning`
**前置依赖**：Phase 2（离线同步完成后再扩展端）

**目标**：扩展到桌面端和电纸书端，验证共享层的跨端复用能力。

**核心交付物**：
- Electron 桌面端项目（复用共享层，独立 UI 层）
- 桌面端 UI（宽屏布局、Tailwind CSS + shadcn/ui）
- 电纸书端 E-Ink 适配主题（去动画、高对比度、大触摸区域、简化布局）
- 三端同步联调通过

**验收标准**：
- 桌面端可正常使用全部功能（快速记录 + 待办 + 日程）
- 电纸书端 UI 适配墨水屏特性，无动画残影
- 三端数据实时同步无冲突（移动端 ↔ 桌面端 ↔ 电纸书端）
- 共享层代码无平台特定逻辑泄漏

---

## Phase 4：产品打磨与扩展

**状态**：`planning`
**前置依赖**：Phase 3

**目标**：提升用户体验，补充辅助功能，逐步完善产品。

**核心交付物**（按优先级排序）：
1. 标签系统完善（跨模块统一标签管理）
2. 全局搜索功能（跨模块搜索日程/待办/快速记录）
3. 数据导出（Markdown / CSV）
4. iOS 打包发布
5. Android Widget 小组件
6. 性能优化与 UI 打磨

**验收标准**：
- 每项交付物独立可验收（具体标准在各 Change 中定义）
