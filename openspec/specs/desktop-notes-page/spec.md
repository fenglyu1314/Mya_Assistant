# 桌面端快速记录页面

> 桌面端快速记录功能的页面和组件规格。

---

### Requirement: 桌面端快速记录页面（NotesPage）
桌面端 SHALL 提供 NotesPage，替换 NotesPlaceholder，使用 `useNotesStore` 展示完整的快速记录功能，包含：

- 顶部内联创建区：多行文本输入框 + 分类选择下拉菜单（灵感 / 备忘 / 日志，默认备忘）+ 提交按钮
- 分类筛选 Tab 栏（全部 / 灵感 / 备忘 / 日志），切换时调用 `setFilter()`
- 快速记录卡片列表（显示内容、分类 Badge、置顶标记、创建时间），置顶记录排在最前
- 每张卡片提供操作：置顶/取消置顶按钮、删除按钮（需确认）
- 空列表时显示空状态提示
- 页面挂载时调用 `fetchNotes()` 和 `subscribeRealtime()`，卸载时取消订阅

#### Scenario: 页面初始加载
- **WHEN** 用户导航到快速记录页面
- **THEN** 显示加载指示器，数据加载完成后展示快速记录卡片列表

#### Scenario: 内联创建快速记录
- **WHEN** 用户在输入框输入内容、选择分类并点击提交
- **THEN** 快速记录被创建到后端，输入框清空，新记录出现在列表中

#### Scenario: 内容为空校验
- **WHEN** 用户未输入内容直接点击提交
- **THEN** 提交按钮禁用或显示提示，不发起请求

#### Scenario: 分类筛选
- **WHEN** 用户点击筛选 Tab 的「灵感」
- **THEN** 列表仅显示 `type` 为 `'idea'` 的记录

#### Scenario: 置顶操作
- **WHEN** 用户点击非置顶记录的置顶按钮
- **THEN** 记录被置顶，列表重新排序，该记录移到顶部

#### Scenario: 删除操作
- **WHEN** 用户点击删除按钮
- **THEN** 显示确认对话框，确认后执行软删除，记录从列表消失

#### Scenario: 实时同步
- **WHEN** 另一端创建或删除了快速记录
- **THEN** 桌面端列表自动更新，无需手动刷新

### Requirement: 桌面端快速记录卡片组件（NoteCard）
桌面端 SHALL 提供 NoteCard 组件，展示单条快速记录：
- 内容文本（多行显示，最大高度限制后截断）
- 分类 Badge（灵感=紫色、备忘=蓝色、日志=绿色）
- 置顶标记（Pin 图标，置顶状态高亮显示）
- 创建时间（相对时间格式，如「5 分钟前」「昨天」）
- 悬停显示操作按钮（置顶、删除）

#### Scenario: 普通记录展示
- **WHEN** 渲染一条分类为「备忘」的非置顶记录
- **THEN** 显示蓝色分类 Badge，置顶图标不高亮

#### Scenario: 置顶记录展示
- **WHEN** 渲染一条置顶记录
- **THEN** 置顶图标高亮显示，卡片视觉上与普通记录有区分

#### Scenario: 悬停交互
- **WHEN** 鼠标悬停在卡片上
- **THEN** 显示置顶和删除操作按钮
